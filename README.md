# @launchstore/shared-puck

> Single source of truth for Puck components and the AI capability registry,
> shared between `launchstore-frontend` (Puck editor), `launchstore-storefront`
> (renderer), and `launchstore` (backend AI registry).

**This is a standalone git repository** (`launchstore-shared` was extracted from
the `launchstore` monorepo on 2026-07-06 to give component versioning its own
release cadence).

## Consumers

| App | What it imports | How |
|---|---|---|
| `launchstore-frontend` | Puck components + fields config + types | `npm` `file:../launchstore-shared` (dev) or `github:launchstore/shared-puck` (CI) |
| `launchstore-storefront` | Puck components (renderer variant) ‚Äî connects to live cart/product APIs | `file:` (dev) or `github:` (CI) |
| `launchstore` (backend) | `dist/registry.json` ‚Äî drives the AI prompt's "what can you build" knowledge | `file:` (dev) or `github:` (CI) |

## Build

```bash
npm install
npm run build
```

Produces:
- `dist/index.js` + `dist/index.d.ts` ‚Äî main entry
- `dist/registry.json` ‚Äî AI capability registry (auto-generated by `scripts/build-registry.ts`)

## Adding a component

1. Create the directory at `src/puck/components/<category>/<Component>/`
2. Add `<component>.meta.ts` with the Puck field schema AND the AI capability descriptor
3. Move the React component + add a `*Client.tsx` variant if it talks to live APIs
4. Re-export from the local `index.ts`
5. `npm run build` ‚Äî registry auto-updates
6. PR title: `feat(shared): add <Component>` ‚Äî fires CI which validates registry shape

See `MIGRATION.md` for the queue of components waiting to land here.

## Adding a new consumer

In the consumer's `package.json`:
```json
{
  "dependencies": {
    "@launchstore/shared-puck": "file:../launchstore-shared"
  }
}
```

In the consumer's `tsconfig.json`:
```json
{
  "compilerOptions": {
    "paths": {
      "@launchstore/shared-puck": ["../launchstore-shared/dist/index.d.ts"],
      "@launchstore/shared-puck/*": ["../launchstore-shared/dist/*"]
    }
  }
}
```

CI swaps `file:` for `github:launchstore/shared-puck#v0.0.x`.

## CI

GitHub Actions at `.github/workflows/ci.yml` ‚Äî runs `npm run build` on every PR
to `main`, fails if `dist/registry.json` shape is invalid (no missing fields, no
duplicate component IDs).

## Versioning

Currently `0.0.1` (WIP). Bump manually in `package.json` when shipping a batch
of components. No semver enforcement yet ‚Äî add `changesets` or `release-please`
when release cadence stabilizes.
